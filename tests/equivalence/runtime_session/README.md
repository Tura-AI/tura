# Runtime / Session equivalence gate

This directory freezes the pre-refactor Runtime and Session behavior at commit
`cdfe91bce2ed4fb62b4dcf276e6ee8ace40264df`. The canonical reference and
inventory were generated by CI run `29696457559` from commit
`76a1f62e665ae131105c9ed73cb7cca3efc9ede1`; the uploaded evidence archive has
SHA-256 `71F22DB14D4F671A4C6FD24C5AA84605F6DA1D9FE3C48BE10491E8217D430B0E`.

The canonical execution environment is the `Backend Business` GitHub Actions
job in `.github/workflows/ci.yml`. It:

1. runs the deterministic candidate capture twice and requires exact byte equality;
2. compares the candidate capture with the committed Phase 0 reference;
3. generates an AST-based test, field, enum variant, and DB-column inventory;
4. runs the repository business suite, including the request/refill raw-byte
   LLM boundary oracle; and
5. uploads all captures and manifests as `phase0-runtime-session-evidence`.

`core-e2e.json` maps the required normal, tool, cancel, timeout, compact,
restart, and busy-input scenarios to their owning tests. Process-level PID,
port, and temporary-path behavior stays in those E2E tests. The deterministic
logical capture uses fixed IDs and timestamps and does not normalize output
after execution.

Do not update a reference to make a behavioral difference pass. A changed
capture requires an explicit compatibility decision and an explanation in the
phase audit.
